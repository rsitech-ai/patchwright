# Support

Patchwright is beta software provided without warranty or a service-level
agreement. Community support is offered on a best-effort basis through
[GitHub Issues](https://github.com/s1korrrr/patchwright/issues). There is no
promise of response time, resolution, compatibility, continued maintenance, or
individual support.

Before opening an issue:

1. Search existing issues and test the latest release or default branch.
2. Run the verification commands in the [README](README.md#build-and-verify).
3. Include the Patchwright version or commit, macOS version, architecture,
   reproduction steps, expected behavior, actual behavior, and relevant
   sanitized logs.
4. Remove access tokens, private keys, credentials, private repository content,
   personal information, and other secrets.

Use [SECURITY.md](SECURITY.md) for suspected vulnerabilities; never report them
in a public support issue. Questions about GitHub, Codex, Xcode, Rust, Apple
Developer accounts, or other third-party products may need to be handled by
their respective providers.

Paid support, hosted services, migration work, and custom deployment are not
part of this open-source beta.
